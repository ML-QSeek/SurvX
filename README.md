# SurvX

> *A shipwrecked child, a deserted island, and an engine that never gives answers.*
>
> *一个船难后的孩子，一座荒岛，一个从不给出答案的引擎。*

**SurvX** 是一个以"约束"替代"奖励"的智能体生存框架。它不提供答案、不提供奖励信号——只执行物理法则，返回客观后果。智能体在约束围成的可行域里，自己发现关系，自己推演方案，自己走出路。

---

## 要解决的问题

让一个系统做出可信的决策，通常有两条路：

- 给它一个奖励函数，让它试错优化——但现实世界付不起试错代价。
- 让它直接给出答案——但它无法为自己的话负责。

SurvX 尝试第三条路：**不给答案，不给奖励。只给约束，只给后果。**

更深一层看，当前主流的仿真与决策架构普遍面临一个结构性问题——三个相互制约的需求无法同时满足：

- 想要更高的推演吞吐量，就要牺牲物理交互的精度；
- 想要更真实的视觉渲染，就要牺牲可同时推演的实体数量；
- 想要快速生成大规模场景，就要简化物理约束的复杂度。

这背后的根源是：这些架构从设计之初，就把"世界规则的演化逻辑"和"智能体的感知决策逻辑"混合在同一套执行链路中。两者互相牵制，牵一发而动全身。

SurvX 的应对方式是**从底层将二者彻底拆开**。世界规则是一张只读的参数表，智能体的认知是另一张动态更新的参数表。两者格式统一，但运行时严格隔离——修改场景的物理约束不需要改动任何智能体配置，反之亦然。这套解耦设计让框架可以同时满足学术研究对"确定性可复现"、"规则可自由配置"、"多智能体可对比实验"的刚性需求。

## 设计

### 三层认知架构

智能体的认知分为三层。下层为上层提供基础，上层在下层之上自然生长。

**本能层**：生理反应与物理边界。速度最快，不经过思考。硬兜底——当认知层的决策触及边界时，本能层直接介入。

**认知层**：因果关系、经验归纳、策略优化。从经历中自动建立实体间的因果依赖——哪些因素影响了哪些结果，影响的程度是多少。通过微型模型和贝叶斯更新持续演化，每次更新都有版本记录，可回溯。

**符号层**：符号化认知与语言表达。当认知丰富到一定程度后自然诞生，不是外部赋予的。内置微型 Transformer，随认知边界同步生长——从几十个符号开始，逐渐扩展词表，逐渐丰富语法。借鉴了语言模型的技术，但完全本地运行，是智能体自己的语言。

三层之上没有"大脑"。外部只提供实体模板和物理法则，不参与决策。

### 六层实体

框架中万物统一建模为 **Entity**，含六层结构。世界实体 5 层（无目标层），智能体实体 6 层（含目标层）。

| 层 | 名称 | 回答的问题 |
|----|------|-----------|
| S | 结构层 | 由什么组成？缺了什么就不再是它？ |
| C | 计算层 | 内部模块之间怎么连接联动？ |
| R | 关系层 | 和这个智能体自身相关的关系——我是谁？ |
| O | 约束层 | 什么阈值越过，实体就崩溃或不再是它自己？ |
| F | 特征层 | 这个实例的具体特征参数值是什么？ |
| G | 目标层 | 希望往哪个方向优化？ |

每个字段带标签：`observed`（世界验证过的）或 `inferred`（猜测，待验证）。inferred 经世界验证后可转为 observed。

结构表只存定义模板，不负责字段注册。字段注册是运行时通过规则推导完成的——用到什么就注册什么，不提前建造整个世界。

### 核心原则

**吝啬世界**：世界从不主动交出完整图纸。只给实体模板，不给实体间的关系。只给后果，不给建议。像一个沉默的物理实验室——你推一下，它告诉你结果，但从不解释为什么。

**事前筛选**：行动方案在诞生前就被约束层过滤。在脑子里筛掉不可行的方案，而不是执行后才发现。一个方案必须先证明自己不会让系统崩溃，才能被考虑。

**偏差即个性**：智能体内部约束与世界物理约束之间的偏差，就是一个智能体的个性来源。偏差过大→死亡。偏差适中→独特的行为风格。存在本身就是检验。

**存在即代价**：不需要外部标注数据来纠错。错误的代价是死亡。正确的回报是继续存在。每一条被保留下来的认知规则，都是用存在换来的。

**关系即网络**：智能体的关系表本质上是一个由经历驱动的、可解释的动态贝叶斯网络。网络结构不是预先构建的，是智能体自己在约束世界里发现的。条件概率不是从海量数据中统计的，是用存在代价单次验证的。每条规则都可以追溯到具体经历。

**最小满足**：实体建模的颗粒度只需细化到能通过关系验证的程度。需要时再展开，不提前建造完整世界。

**世界固定，认知学习**：物理法则固定不变，智能体的个人认知从经历中增量学习、版本可回溯。两者格式统一，语义分离。

**规则推导**：实体发现、空间注册、字段访问全部通过规则动态推导，不硬编码。

### 数据流

每个 tick 执行以下流程：
序列快照（保存旧状态）
↓
深拷贝旧状态
↓
空间注册（动态推导所有实体的空间关系）
↓
目标层动态生成
↓
意图生成 → 候选行动方案
↓
约束预筛选（排除不可行方案）
↓
模型推演（预测每个方案的后果）
↓
多目标评估 → 选定最终方案
↓
执行行动 → 收集反馈
↓
反馈表 → 临时表 → 关系表
↓
关系表 × 参数表/模型库 → 计算变化量
↓
变化量叠加到新状态
↓
约束检查（越界即死亡或触发响应）
↓
写入新状态 → 进入下一 tick

反馈表记录当前有哪些活跃字段在起作用。关系表将反馈归并为"哪个目标字段被哪些源字段影响"。关系表配合参数表（或模型库）计算每个字段的变化量，叠加到状态，形成新状态，新状态产生下一轮反馈。这个闭环是框架运行的骨架。

### 两类规则

SurvX 区分两种规则：

- **参数表**：世界物理法则。固定不变，初始化后不更新。
- **模型库**：智能体个人的认知参数。从经历中增量学习，版本可回溯，随时间演化。

推演时，世界规则走参数表，智能体认知走模型库。两者格式相同，通过统一预测接口调用。是否区分、何时从参数表切换到模型库，由智能体的认知状态决定。

### 学术脉络

SurvX 处理的核心问题可以归结为：**在约束集合内寻找可行的演化路径。系统不追求最优解，只追求持续不越界。** 这区别于基于奖励函数的最优策略搜索——这里的目标不是最大化什么，而是持续满足约束条件。约束本身是分层的（物理约束、生理约束、认知约束），且可以在经历失败后动态增生——每次越界都可能产生新的约束。多个智能体共存时，各自的约束相互耦合，形成群体层面的可行域。

### 四层产出

当一个智能体走完一生（或一批智能体各自走完一生），框架产出四个层次的结果：

| 层级 | 产出 | 说明 |
|------|------|------|
| 叙事层 | 人生故事 | 该智能体的第一人称经历记录 |
| 关系层 | 纯关系表 | 剥离个性和即时状态，只留因果规则网络 |
| 个性层 | 带性格的判断模型 | 同一套关系表 + 不同性格参数 → 不同决策风格 |
| 群体层 | 多智能体合并关系表 + 性格库 | 去重、加权、冲突消解后的群体认知 |

不同智能体可以共享同一套关系表但表现出不同的行为——谨慎者在中毒概率 20% 时不吃红果子，冒险者在饥饿时愿意尝试。合并多个智能体的关系表后，形成多风格的判断模型。

个体消失后，关系表可以留下来。后人在此基础上继续发现、修正、扩展。

## Demo：鲁滨逊在荒岛

> 这是 SurvX 框架的一个验证性示例。框架支持多种环境和智能体配置，鲁滨逊是第一个可运行的 Demo。

### 设定

鲁滨逊，7 岁，有简单英语能力。南太平洋火山岛，约 5 平方公里。初始饥饿度 75，体力 70，位置海滩。

**世界常量**（每 tick 固定执行，构成环境的基础节律）：

| 常量 | 规则 |
|------|------|
| 耗散 | 体力每 tick -0.1 |
| 昼夜 | 12 tick 交替 |
| 潮汐 | 6 tick 涨落 |
| 风 | 风力 3，风向 [1,0,0] |

**环境实体**按五层模板定义：

| 层 | 回答的问题 | 示例（椰子） |
|----|-----------|-------------|
| S 结构层 | 由什么组成？ | 外壳、椰肉、椰汁 |
| C 计算层 | 内部怎么联动？ | 打开外壳→获得椰肉和椰汁 |
| R 关系层 | 实体间常见关系 | 长在椰树上，掉落后在地面 |
| O 约束层 | 崩溃阈值 | 从 >10 米掉落→外壳破裂 |
| F 特征层 | 典型参数范围 | 重量 500-1500g，椰汁 200-400ml |

智能体不被告知这些信息，只能通过有限观测逐步发现。世界是吝啬的——它从不主动交出完整图纸。

### 外部交互

环境中有一个"父亲"角色，三种状态：

| 状态 | 行为 |
|------|------|
| 在家 | 鲁滨逊可主动提问，父亲选答或不答 |
| 出门了 | 鲁滨逊只能自己解决，问题积累到待问列表 |
| 回来了 | 可补问积累的问题 |

父亲教的知识带信任度标签（初始 0.9），需经世界验证后动态调整。父亲不说话时即旁观模式。

### 当前可验证的核心能力

在已闭合的最简闭环中，可以直观验证框架的以下特性：

**确定性与可复现性**：给定完全相同的初始状态和随机种子，多次运行得到完全一致的状态演化轨迹。每一步状态变化都有完整的快照记录，可以从任意 tick 精准复盘。

**规则解耦**：修改世界常量（如体力衰减速率从 -0.1 改为 -0.05）或调整约束阈值（如死亡线从 0 改为 5），不需要改动任何智能体认知配置代码。反之亦然。

**状态可追溯**：引擎每 tick 保存完整的状态快照和变更日志。任何时刻中断运行后，可以从最近的快照精确恢复场景，继续推演。

**观测透明**：内置仪表盘实时展示时序图表、状态快照和原始数据，所有字段的变化路径一目了然。
